## 1. 训练损失指标 (Train Loss)

### 1.1 总训练损失 (Train Loss)

```
Train Loss: 0.391096
```

**含义**：所有损失项的加权总和，代表模型在训练集上的整体误差。

**计算公式**：
```python
Train Loss = λ_l1 × L1 + λ_ssim × SSIM_loss + λ_freq × Freq + λ_grad × Grad
           = 1.0 × L1 + 0.1 × (1-SSIM) + 0.01 × Freq + 0.05 × Grad
```

**健康范围**：
- 🟢 优秀：< 0.2
- 🟡 良好：0.2 - 0.5
- 🟠 一般：0.5 - 1.0
- 🔴 较差：> 1.0

**理想趋势**：持续稳定下降，最终收敛到 0.1-0.3 范围。

---

### 1.2 L1 损失 (MAE)

```
l1: 0.136795
```

**含义**：像素级平均绝对误差 (Mean Absolute Error)，衡量重建图像与真实图像在像素级别的差异。

**计算公式**：
```python
L1 = mean(|SR - HR|)
```

**健康范围**：
- 🟢 优秀：< 0.05
- 🟡 良好：0.05 - 0.10
- 🟠 一般：0.10 - 0.15
- 🔴 较差：> 0.15

**权重配置**：`lambda_l1 = 1.0`（主导损失项）

**注意事项**：
- L1 损失对异常值不敏感，训练更稳定
- 过低的 L1（<0.01）可能导致图像过度平滑

---

### 1.3 SSIM 损失

```
ssim: 0.885031
```

**⚠️ 重要说明**：代码中显示的 `ssim` 实际上是 **SSIM Loss = 1 - SSIM**，而非 SSIM 值本身！

**真实 SSIM 值计算**：
```python
真实 SSIM = 1 - ssim_loss = 1 - 0.885031 = 0.114969 (约 11.5%)
```

**含义**：结构相似性损失，衡量图像亮度、对比度和结构的综合相似度。

**计算公式**：
```python
SSIM_loss = 1 - SSIM(SR, HR)
SSIM = (2μ_xμ_y + c1)(2σ_xy + c2) / ((μ_x² + μ_y² + c1)(σ_x² + σ_y² + c2))
```

**健康范围**（SSIM Loss）：
- 🟢 优秀：< 0.10（真实 SSIM > 0.90）
- 🟡 良好：0.10 - 0.20（真实 SSIM 0.80-0.90）
- 🟠 一般：0.20 - 0.30（真实 SSIM 0.70-0.80）
- 🔴 较差：> 0.30（真实 SSIM < 0.70）

**权重配置**：`lambda_ssim = 0.1`（降低权重避免梯度主导）

---

### 1.4 频域损失 (Frequency Loss)

```
freq: 13.918727
```

**含义**：傅里叶频域差异，衡量图像在频域上的重建质量，对纹理细节敏感。

**计算公式**：
```python
Freq = ||FFT(SR) - FFT(HR)||_1
```

**健康范围**：
- 🟢 优秀：< 5
- 🟡 良好：5 - 10
- 🟠 一般：10 - 20
- 🔴 较差：> 20

**权重配置**：`lambda_freq = 0.01`（大幅降低避免数值主导）

**注意事项**：
- 频域损失数值通常较大，需要较小权重
- 对高频细节（边缘、纹理）重建很重要

---

### 1.5 梯度损失 (Gradient Loss)

```
grad: 0.532217
```

**含义**：图像梯度差异，衡量边缘清晰度和细节保持能力。

**计算公式**：
```python
Grad = ||∇(SR) - ∇(HR)||_1
# 使用 Sobel 算子或简单差分计算梯度
```

**健康范围**：
- 🟢 优秀：< 0.2
- 🟡 良好：0.2 - 0.4
- 🟠 一般：0.4 - 0.6
- 🔴 较差：> 0.6

**权重配置**：`lambda_grad = 0.05`

## 2. 验证损失指标 (Validation Loss)

### 2.1 总验证损失 (Val Loss)

```
Val Loss: 0.519720
```

**含义**：模型在验证集上的总损失，用于检测过拟合。

**健康标准**：
- Val Loss 应略高于 Train Loss（差距 < 20%）
- 差距 > 50% 说明严重过拟合
- Val Loss 持续上升说明开始过拟合

**示例分析**：
```
Train Loss: 0.391
Val Loss:   0.520  (差距 33%，轻微过拟合迹象)
```

## 3. 评估指标 (Evaluation Metrics)

### 3.1 PSNR (峰值信噪比)

```
Val PSNR: 12.63 dB
```

**含义**：Peak Signal-to-Noise Ratio，衡量重建图像质量的客观指标，基于像素级均方误差。

**计算公式**：
```python
PSNR = 10 × log10(MAX_I² / MSE)
     = 20 × log10(MAX_I / sqrt(MSE))

其中：
- MAX_I = 1.0 (归一化图像最大值)
- MSE = mean((SR - HR)²)
```

**质量等级**：

| 等级 | PSNR 范围 | 评价 | 说明 |
|------|-----------|------|------|
| 🟢 优秀 | > 35 dB | 接近原图 | 肉眼难以区分差异 |
| 🟢 良好 | 30-35 dB | 高质量 | 轻微差异，可接受 |
| 🟡 一般 | 25-30 dB | 中等质量 | 可见轻微模糊/噪声 |
| 🟠 较差 | 20-25 dB | 低质量 | 明显失真 |
| 🔴 很差 | 15-20 dB | 不可用 | 严重模糊/伪影 |
| ⚫ 极差 | < 15 dB | 完全失败 | 模型未学到任何特征 |

**对应 MSE 参考**：

| PSNR | MSE | 质量 |
|------|-----|------|
| 40 dB | 0.0001 | 完美 |
| 30 dB | 0.001 | 优秀 |
| 20 dB | 0.01 | 较差 |
| 12 dB | 0.04 | 极差 |

**问题诊断**：
- PSNR < 20 dB：检查数据加载是否正确
- PSNR < 15 dB：模型完全未训练，检查损失函数和学习率

---

### 3.2 SSIM (结构相似性指数)

```
Val SSIM: 0.0783
```

**含义**：Structural Similarity Index，衡量图像结构信息保持程度的感知指标。

**计算公式**：
```python
SSIM(x, y) = [l(x,y)]^α × [c(x,y)]^β × [s(x,y)]^γ

亮度项：l(x,y) = (2μ_xμ_y + c1) / (μ_x² + μ_y² + c1)
对比度项：c(x,y) = (2σ_xσ_y + c2) / (σ_x² + σ_y² + c2)
结构项：s(x,y) = (σ_xy + c3) / (σ_xσ_y + c3)
```

**质量等级**：

| 等级 | SSIM 范围 | 评价 |
|------|-----------|------|
| 🟢 优秀 | > 0.90 | 结构保持极好 |
| 🟢 良好 | 0.80-0.90 | 结构保持较好 |
| 🟡 一般 | 0.70-0.80 | 有一定结构损失 |
| 🟠 较差 | 0.50-0.70 | 结构损失严重 |
| 🔴 很差 | 0.30-0.50 | 结构模糊 |
| ⚫ 极差 | < 0.30 | 结构完全丢失 |

**与 PSNR 的区别**：
- PSNR：基于像素误差，对位置偏移敏感
- SSIM：基于结构信息，更符合人眼感知

**问题诊断**：
- SSIM < 0.5 但 PSNR > 25：图像过度平滑
- SSIM < 0.2：模型未学到有效特征或验证集错误

---

### 3.3 RMSE (均方根误差)

**含义**：Root Mean Square Error，L2 范数的像素级误差。

**计算公式**：
```python
RMSE = sqrt(mean((SR - HR)²))
```

**与 PSNR 的关系**：
```python
PSNR = 20 × log10(1.0 / RMSE)
```

**特点**：
- 对异常值（outliers）比 L1 更敏感
- 单位与像素值相同，直观可解释

## 4. 学习率指标 (Learning Rate)

```
LR: 5.00e-05
```

### 4.1 调度策略：Warmup + Cosine Annealing

**配置参数**：
```python
base_lr = 5e-5
warmup_epochs = 10
min_lr = 1e-7
total_epochs = 200
```

### 4.2 Warmup 阶段 (Epoch 1-10)

```python
lr = base_lr × (current_epoch / warmup_epochs)
```

| Epoch | 学习率 | 说明 |
|-------|--------|------|
| 1 | 5e-6 | 极小，稳定初始化 |
| 5 | 2.5e-5 | 逐渐增加 |
| 10 | 5e-5 | 达到基准值 |

**作用**：防止初始阶段梯度爆炸，稳定训练早期。

### 4.3 Cosine Annealing 阶段 (Epoch 11-200)

```python
progress = (epoch - 10) / (200 - 10)
lr = min_lr + (base_lr - min_lr) × 0.5 × (1 + cos(π × progress))
```

| Epoch | 学习率 | 说明 |
|-------|--------|------|
| 11 | ~5e-5 | 开始衰减 |
| 50 | ~4e-5 | 缓慢下降 |
| 100 | ~2.5e-5 | 中期 |
| 150 | ~1e-5 | 精细调整 |
| 200 | 1e-7 | 最小值 |

**作用**：帮助模型跳出局部最优，精细收敛。

## 5. 早停指标 (Early Stopping)

### 5.1 配置参数

```python
patience = 15          # 容忍验证无提升的次数
metric = 'psnr'        # 监控指标
mode = 'max'           # 最大化 PSNR
min_delta = 0.01       # 最小提升阈值 (0.01 dB)
```

### 5.2 状态指标

```
EarlyStopping: 3/15
```

| 指标 | 含义 |
|------|------|
| counter | 已连续无提升的验证次数 |
| patience | 最大容忍次数 |
| best_value | 历史最佳 PSNR 值 |

### 5.3 触发条件

```python
if 当前 PSNR ≤ 最佳 PSNR + 0.01:
    counter += 1
    if counter >= 15:
        早停触发，结束训练
else:
    counter = 0
    更新最佳值
```

## 8. 常见问题诊断

### Q1: Train Loss 正常但 Val PSNR 极低 (< 15 dB)

**可能原因**：
1. 验证集路径错误，加载了未配对的图像
2. 验证集标签是全黑/全白图像
3. 预处理方式不一致（如训练集归一化到[0,1]，验证集到[-1,1]）

**诊断命令**：
```python
# 检查验证集样本
val_sample = next(iter(val_loader))
print(f"Input range: [{val_sample[0].min():.3f}, {val_sample[0].max():.3f}]")
print(f"Target range: [{val_sample[1].min():.3f}, {val_sample[1].max():.3f}]")
```

### Q2: SSIM Loss 很高 (> 0.9)

**说明**：真实 SSIM < 0.1，模型输出接近噪声。

**解决方案**：
1. 降低 SSIM 损失权重
2. 先只用 L1 损失训练若干 epoch 预热
3. 检查输入输出通道是否正确

### Q3: Freq Loss 主导训练

**现象**：Freq >> L1 + SSIM + Grad

**解决方案**：
```python
# 已优化配置
lambda_freq = 0.01  # 大幅降低
```

### Q4: 训练不稳定 (Loss 震荡)

**可能原因**：
1. 学习率过大
2. Batch Size 过小
3. 梯度爆炸

**解决方案**：
```python
# 已配置
grad_clip = 1.0     # 梯度裁剪
batch_size = 4      # 较小但稳定的 batch
lr = 5e-5          # 较低的学习率
```

## 11. 总结

| 核心指标 | 目标值 | 最低可接受 | 检查频率 |
|----------|--------|------------|----------|
| Train Loss | < 0.2 | < 0.5 | 每 epoch |
| Val PSNR | > 30 dB | > 25 dB | 每 5 epoch |
| Val SSIM | > 0.90 | > 0.80 | 每 5 epoch |
| Train-Val Gap | < 20% | < 50% | 每 5 epoch |

**关键原则**：
1. **PSNR < 20 dB**：一定是数据或模型配置有问题
2. **SSIM < 0.5**：模型未学到有效特征
3. **Val Loss >> Train Loss**：过拟合，需增加正则化
4. **所有指标不提升**：学习率可能过小或模型容量不足
